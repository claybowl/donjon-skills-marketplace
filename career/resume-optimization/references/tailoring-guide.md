# Resume Tailoring Guide

Strategies for adapting your resume to match specific job descriptions while maintaining authenticity.

## The 80/20 Tailoring Rule

**80% Base Resume:** Core experience, skills, and achievements that apply broadly

**20% Customization:** Targeted adjustments for each specific application

This approach balances efficiency (not rewriting from scratch) with effectiveness (showing fit for each role).

## What to Customize

### 1. Professional Summary (If Present)

**Customize:**

- Opening that mirrors role title or specialty
- Key technologies from job requirements
- Achievement that aligns with company priorities

**Example Base:**
> Senior Software Engineer with 7 years building scalable backend systems.

**Tailored for FinTech:**
> Senior Software Engineer with 7 years building high-scale payment and financial systems with strict compliance requirements.

**Tailored for Healthcare:**
> Senior Software Engineer with 7 years building secure, HIPAA-compliant healthcare platforms.

### 2. Skills Section Order

**Reorder to prioritize:**

1. Required skills from job description (first)
2. Preferred skills mentioned (second)
3. Other relevant skills (third)

**Example:**

Job requires: Python, AWS, PostgreSQL
Job prefers: Kubernetes, Terraform

**Before (alphabetical):**

```text
AWS, Docker, Go, Java, Kubernetes, PostgreSQL, Python, Terraform
```

**After (prioritized):**

```text
Python, AWS, PostgreSQL, Kubernetes, Terraform, Docker, Java, Go
```

### 3. Experience Bullet Selection

**Choose bullets that:**

- Demonstrate required skills
- Match requested responsibilities
- Show relevant industry experience
- Highlight appropriate scope/scale

**Strategy:**

- Lead with most relevant bullets
- Include 3-5 bullets per recent role
- Can swap bullets between applications

### 4. Achievement Emphasis

**Adjust emphasis based on role:**

**For a Technical IC role:**
Lead with technical complexity, scale, performance

**For a Tech Lead role:**
Lead with leadership, mentoring, cross-team coordination

**For an Architect role:**
Lead with system design, decision-making, long-term impact

### 5. Keywords and Terminology

**Mirror exact language:**

- Job says "microservices" → use "microservices" not "distributed systems"
- Job says "React" → use "React" not "frontend framework"
- Job says "Agile" → use "Agile" not "iterative methodology"

## Step-by-Step Tailoring Process

### Step 1: Analyze the Job Description

**Read thoroughly and highlight:**

- Required qualifications (must-haves)
- Preferred qualifications (nice-to-haves)
- Key responsibilities
- Technologies and tools mentioned
- Soft skills emphasized
- Company values or culture indicators

### Step 2: Map Your Experience

**For each requirement, identify:**

- Which of your experiences demonstrates this?
- What specific accomplishments prove this skill?
- What metrics or outcomes can you cite?

**Create a mapping:**

```text
Requirement: "Experience with distributed systems"
My evidence: Built payment processing system handling 1M daily transactions
Bullet: "Architected distributed payment system processing 1M daily transactions..."
```

### Step 3: Adjust Your Summary

**Incorporate:**

- Role title or specialty alignment
- 1-2 key required technologies
- Relevant industry or domain reference

### Step 4: Reorder Skills Section

**Prioritize:**

1. All required skills (even if you're less strong)
2. Preferred skills you have
3. Related skills that show breadth

### Step 5: Select and Reorder Bullets

**For each recent role:**

- Lead with bullet most relevant to new role
- Include bullets demonstrating required skills
- Remove or demote less relevant bullets
- Adjust language to match job description

### Step 6: Review for Keyword Coverage

**Checklist:**

- [ ] All required skills mentioned somewhere
- [ ] Key technologies appear in Skills and Experience
- [ ] Soft skills demonstrated in bullets (not just listed)
- [ ] Industry terminology matches job description

## Tailoring by Role Type

### Backend/Platform Engineer

**Emphasize:**

- API design and development
- Database design and optimization
- System scalability and performance
- Infrastructure and deployment

**Adjust bullets to highlight:**

- Request handling volumes
- Latency improvements
- Database query optimization
- System reliability metrics

### Frontend Engineer

**Emphasize:**

- User interface development
- Frontend frameworks and libraries
- Performance optimization
- Accessibility and UX

**Adjust bullets to highlight:**

- User-facing features
- Page load improvements
- Responsive design
- Cross-browser compatibility

### Full-Stack Engineer

**Emphasize:**

- End-to-end feature development
- Both frontend and backend skills
- Integration expertise
- Versatility

**Balance bullets between:**

- Frontend achievements
- Backend achievements
- Full-stack project ownership

### DevOps/SRE

**Emphasize:**

- Infrastructure automation
- CI/CD pipelines
- Monitoring and observability
- Incident response

**Adjust bullets to highlight:**

- Automation percentages
- Deployment frequency
- Uptime metrics
- Incident resolution times

### Engineering Manager/Tech Lead

**Emphasize:**

- Team leadership
- Project delivery
- Mentoring and growth
- Cross-functional collaboration

**Adjust bullets to highlight:**

- Team size and scope
- Project outcomes
- Reports' growth/promotions
- Stakeholder management

## Tailoring by Company Type

### Startup (Early Stage)

**Emphasize:**

- Versatility and broad skills
- Shipping quickly
- Working with ambiguity
- Founding/early team experience

**Language adjustments:**

- "Shipped" over "Delivered"
- "Built from scratch" over "Maintained"
- "Wore multiple hats" if applicable

### Scale-up (Growth Stage)

**Emphasize:**

- Scaling systems
- Building processes
- Growing teams
- Maintaining velocity with growth

**Language adjustments:**

- Scale metrics (10x, 100x growth)
- Team building and hiring
- Process establishment

### Enterprise

**Emphasize:**

- Large-scale systems
- Cross-team coordination
- Compliance and security
- Long-term maintenance

**Language adjustments:**

- Enterprise-scale numbers
- Governance and compliance
- Stakeholder management
- Legacy system experience

### FAANG/Big Tech

**Emphasize:**

- Large-scale distributed systems
- Data-intensive applications
- Technical excellence
- Measurable impact

**Language adjustments:**

- Specific scale metrics
- Company-relevant experience
- Technical depth

## Common Tailoring Mistakes

### Over-Tailoring

**Problem:** Resume sounds like you wrote it for this one job

- Too many exact phrases copied from JD
- Unnatural language
- Claims that don't match your real experience

**Fix:** Use your own words that convey the same skills

### Under-Tailoring

**Problem:** Generic resume that doesn't show fit

- No keywords from job description
- No alignment with role requirements
- One-size-fits-all bullets

**Fix:** Spend 15-30 minutes customizing for each application

### Keyword Stuffing

**Problem:** Resume reads like a keyword list

- Unnatural repetition
- Skills without context
- Bullets that exist just for keywords

**Fix:** Integrate keywords naturally into achievement bullets

### Misrepresenting Experience

**Problem:** Claiming expertise you don't have

- Listing skills you barely touched
- Overstating your role or impact
- Making up metrics

**Fix:** Stay honest; emphasize what you CAN do authentically

## Quick Tailoring Checklist

### Before Submitting

1. [ ] Read job description completely
2. [ ] Summary aligns with role (if present)
3. [ ] Skills section prioritizes required skills
4. [ ] Top bullets per role are most relevant
5. [ ] Key technologies mentioned in Experience
6. [ ] Soft skills demonstrated (not just listed)
7. [ ] Company/industry fit shown if possible
8. [ ] File renamed appropriately
9. [ ] PDF exported fresh after changes
10. [ ] Quick proofread after modifications

## Version Control Strategy

### Organizing Tailored Versions

**Folder structure:**

```text
Resumes/
├── Base_Resume_2024.docx
├── Applications/
│   ├── Stripe_Senior_SWE_Dec2024.pdf
│   ├── Datadog_Backend_Dec2024.pdf
│   └── Notion_Fullstack_Dec2024.pdf
└── Templates/
    ├── Template_Backend.docx
    ├── Template_Frontend.docx
    └── Template_Manager.docx
```

### Tracking Applications

Simple spreadsheet:

| Company | Role | Date Applied | Version | Status | Notes |
| ------- | ---- | ------------ | ------- | ------ | ----- |
| Stripe | Senior SWE | 2024-12-15 | Backend v3 | Applied | Referred by John |
| Datadog | Backend | 2024-12-16 | Backend v3 | Screen | Call 12/20 |

### When to Create New Templates

Create role-type templates when you:

- Apply to 3+ similar roles
- See recurring requirement patterns
- Want to speed up future applications

Templates should be 90% complete, needing only company-specific tweaks.
