# Report Structure Reference

This document defines the canonical structure for each Donjon Intel Report type.
Use it as a skeleton — adapt sections based on what the research actually reveals.
Not every report needs every section. But every report needs the core spine.

## Core Spine (All Reports)

Every report includes these elements, in this order:

1. **Cover / Title Block**
   - Title (large, left-aligned)
   - Subtitle with report type and focus areas
   - Metadata table: Subject, Scope, Prepared For, Prepared By, Report Date,
     Classification, Report #
   - Optional: Interview Date, Series Name

2. **Executive Read**
   - 3-4 paragraphs maximum
   - First sentence = most important finding
   - Bottom-line callout box at the end
   - Must stand alone — someone reading only this section gets the core message

3. **Main Analytical Sections** (varies by report type — see below)

4. **What This Means for [Client]**
   - Numbered strategic insights (3-5)
   - Each insight = what it is + what to do about it
   - Directly references the client by name

5. **Confidence Scores**
   - Table with columns: Conclusion, Confidence, Basis
   - 6-10 most important conclusions
   - Honest about uncertainty

6. **Source Base**
   - Bulleted list of every source referenced
   - Include source type (e.g., "IRS 990-PF filing", "LinkedIn profile",
     "Texas Tribune investigative piece")

7. **Signoff**
   - "Prepared with care by Donjon Intelligence Systems"
   - Name and title
   - Date

## Report Type: Organizational Intelligence

Use when: Deep dive on a company, school, organization, or movement.

### Structure

- Cover
- Executive Read
- **Part I — What It Is**
  - Origin story and founding
  - Mission, model, philosophy
  - Current operations and scale
- **Part II — Who Runs It**
  - Leadership profiles
  - Board / advisors
  - Key personnel roster table
- **Part III — How It's Funded**
  - Revenue model
  - Investors / donors
  - Financial health indicators
- **Part IV — The Ecosystem**
  - Partners, competitors, adjacent organizations
  - Regulatory environment
  - Market position
- **Part V — Risks and Red Flags** (if applicable)
  - Legal issues
  - Controversies
  - Financial concerns
- What This Means for [Client]
- Confidence Scores
- Source Base
- Signoff

## Report Type: Personnel Dossier

Use when: Background research on a specific person.

### Structure

- Cover
- Executive Read (frame why this person matters)
- **Fast Facts Table**
  - Full name, credentials, current title, location, prior roles,
    recognition, specialties, platforms, speaking
- **Career Trajectory**
  - Chronological sections, each with:
    - What they did
    - Why it matters
    - Pattern match callout for the client
- **Why This Figure Matters for [Client]**
  - Insights table: What they signal → What it tells us → How client responds
- **Network / Connections** (if relevant)
  - Board memberships
  - Investment relationships
  - Professional affiliations
- Confidence Scores
- Source Base
- Signoff

## Report Type: Deep Report

Use when: Follow-the-money, power mapping, ecosystem analysis, political landscape.

### Structure

- Cover (classification: "Client Use Only — Deep Report")
- Executive Read
- **Part I — The Subject**
  - Deep exploration of what the thing actually is
  - Philosophy, methodology, history
  - This section should be thorough enough to stand as its own primer
- **Part II — Follow the Money**
  - Tier 1: Primary funding source (table: entity, role, scale)
  - Tier 2: Secondary funding (commercialization arms, spinoffs)
  - Tier 3: Broader ecosystem (VC, philanthropy, policy)
  - Channel breakdowns: Venture Capital, Billionaire Philanthropy, Policy/Legislation
- **Part III — The Infrastructure**
  - Platforms, networks, accelerators that make scale possible
  - Table: platform, model, scale, funding
  - Market size and growth trajectory
- **Part IV — The Power Map**
  - Inner Ring: Core decision-makers (table: person, position, why they matter)
  - Middle Ring: Investors and enablers
  - Outer Ring: Political and policy operators
- **Part V — How the Money Flows**
  - Layer-by-layer sequence: Capital Formation → Entity Creation →
    Policy Infrastructure → Revenue and Scale
  - "The loop" callout box explaining the complete cycle
- What This Means for [Client]
- Confidence Scores
- Source Base
- Signoff

## Report Type: Interview Intelligence

Use when: Preparing someone for a job interview.

### Structure

- Cover
- Executive Read (the organization + what to expect)
- **The Organization** (condensed org intel — 3-4 pages max)
  - What they do
  - Who leads them
  - Culture and values
  - Recent developments
- **The Interviewers** (personnel dossiers on who they'll meet)
  - Fast facts
  - Background
  - What they likely care about
- **The Role** (what the job actually requires)
  - Official description vs. reality
  - What success looks like
  - Common mistakes candidates make
- **Interview Strategy**
  - Positioning adjustments
  - Key phrases to use / avoid
  - Questions to ask
  - 90-day plan framework (if applicable)
- **Prep Timeline** (countdown to interview day)
  - Day-by-day checklist
- Confidence Scores
- Source Base
- Signoff

## Report Type: Competitive Intelligence

Use when: Market landscape, competitor analysis.

### Structure

- Cover
- Executive Read
- **Market Overview**
  - Size, growth, key trends
  - Segmentation
- **Competitor Profiles** (one subsection per competitor)
  - What they do
  - Strengths / weaknesses
  - Funding and financials
  - Key personnel
- **Comparison Matrix**
  - Feature-by-feature or capability-by-capability table
- **Positioning Analysis**
  - Where gaps exist
  - Differentiation opportunities
  - Threats to monitor
- What This Means for [Client]
- Confidence Scores
- Source Base
- Signoff

## Report Type: Due Diligence

Use when: Evaluating an investment, partnership, or acquisition target.

### Structure

- Cover (classification: "Client Use Only — Due Diligence")
- Executive Read (go/no-go framing)
- **Entity Overview**
  - Legal structure
  - History
  - Current operations
- **Financial Analysis**
  - Revenue, burn rate, runway
  - Funding history
  - Cap table (if available)
- **Leadership Assessment**
  - Founder/exec backgrounds
  - Track record
  - Red flags
- **Market Position**
  - Competitors
  - Moat / defensibility
  - Growth trajectory
- **Risk Register**
  - Legal risks
  - Financial risks
  - Operational risks
  - Reputational risks
  - Table format: Risk, Likelihood, Impact, Mitigation
- **Recommendation**
  - Clear go/no-go/conditional recommendation
  - Key conditions or due diligence items remaining
- Confidence Scores
- Source Base
- Signoff

## Page Break Strategy

Force page breaks before:
- Each major "Part" section
- The Confidence Scores section
- Between distinct analytical sections that change topic completely

Don't force page breaks:
- Between subsections within the same Part
- Before every table (let them flow naturally)
- Before callout boxes (they should feel connected to the text above)

## Length Guidelines

- Don't pad. A 6-page report that's all substance beats a 20-page report with fluff.
- But don't skimp. If the research reveals 20 pages of genuine intelligence, use 20 pages.
- The Executive Read is never longer than one page.
- Tables should be concise — if a table has more than 10 rows, consider whether it
  needs to be a table or whether you're better served by a narrative summary.
